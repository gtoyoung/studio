# **App Name**: Lunch Polls

## Core Features:

- Daily Poll Creation: Automatically create a new poll every day to ask users about their lunch participation.
- Participation Recording: Allow users to easily indicate whether they will participate in lunch that day.
- Real-time Statistics: Display real-time statistics on lunch participation, showing how many people have confirmed or declined.
- Historical Data Storage: Store historical poll data for generating reports and tracking trends over time, using Firestore.
- Reporting: Provide insights on average participation rates, popular days, and potential cost savings based on participation.

## Style Guidelines:

- Primary color: HSL 210, 64%, 51% (RGB: #3D92F4) to inspire productivity and group activity.
- Background color: HSL 210, 20%, 95% (RGB: #F0F4F8). A very light tint of the primary.
- Accent color: HSL 180, 58%, 44% (RGB: #2DBDBD) for important interactions, like confirmations.
- Body and headline font: 'PT Sans', a humanist sans-serif for a modern yet welcoming feel.
- Use clear, simple icons to represent participation, non-participation, and statistical insights.
- Design a clean and straightforward layout for quick daily interactions. Keep it very simple for the user to understand the information and participate
- Employ subtle transitions and animations to provide feedback on user interactions and guide users through the interface.
